# Contributing to Obsidian Collaborative Plugin

We welcome contributions! Please follow these guidelines to ensure a smooth process.

## Getting Started

1.  Fork the repository.
2.  Clone your fork locally.
3.  Install dependencies using `npm install`.

## Development Guidelines

- **Monorepo**: We use npm workspaces. Run commands from the root using `--workspace=<workspace-name>`.
- **TypeScript**: All packages are written in TypeScript. Ensure no type errors before committing.
- **Code Style**: Follow the existing code style. Prettier configuration is forthcoming.

## Pull Request Process

1.  Create a new branch for your feature or fix.
2.  Commit your changes with descriptive messages.
3.  Push to your fork and submit a Pull Request.
4.  Describe your changes in the PR description.

## Reporting Issues

If you encounter bugs, please open an issue in the repository with details on how to reproduce it.
